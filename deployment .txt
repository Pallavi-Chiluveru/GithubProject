To deploy your Collaborative Git Platform in production, you will need a combination of platforms that support static assets, backend computing, and—most importantly—persistent file storage for your Gitea engine.

Since Git repositories are actual physical directories and database files (when using SQLite), deploying Gitea on standard ephemeral servers like standard Render/Heroku dynos would cause all user repositories to be permanently deleted every time the server restarts.

Below is the step-by-step guide on where and how to deploy each service (including Gitea) and the exact URL changes you need to make in your code environment.

🗺️ The Recommended Stack
Frontend (React/Vite): Deployed on Vercel (Free, fast, with SSL).
Backend (Express + WebSockets): Deployed on Render (Free Web Service tier, supports live sockets).
Database (MongoDB): Deployed on MongoDB Atlas (Free cloud database tier).
Git Engine (Gitea): Deployed on Railway (Recommended because it has built-in one-click Gitea templates, offers persistent storage volumes, and is very affordable/free-tier friendly).
🛠️ Step 1: Deploy Gitea (The Git Engine) on Railway
Railway is the easiest and most reliable PaaS to host Gitea with a persistent disk.

Create a free account on Railway.app.
Click New Project in your dashboard.
In the search bar, search for Gitea (or select "Deploy from template" and select the official Gitea template).
Click Deploy. Railway will provision:
A Gitea container.
A PostgreSQL database (or persistent SQLite volume).
A persistent disk volume (mounted at /data) so your Git repositories are never lost when Gitea restarts.
Once deployed, Railway will assign you a public URL (e.g., https://gitea-production.up.railway.app). Keep this URL handy.
🗄️ Step 2: Deploy MongoDB Atlas (The Database)
Register or log in at MongoDB Atlas.
Create a new Shared Cluster (which is completely free).
Under Security > Database Access, create a user (e.g., db_user) and a secure password.
Under Network Access, click Add IP Address and choose Allow Access from Anywhere (0.0.0.0/0) so Render's backend can reach your database.
In the Database tab, click Connect > Drivers to copy your MongoDB Connection String. It will look like this: mongodb+srv://db_user:<password>@cluster0.xxxx.mongodb.net/?retryWrites=true&w=majority (Replace <password> with your database user's password).
🖥️ Step 3: Deploy the Backend on Render
Log into Render.com.
Click New + and select Web Service.
Connect your GitHub repository.
Fill in the following settings:
Name: git-platform-backend
Root Directory: backend
Language: Node
Build Command: npm install
Start Command: node server.js
Render will assign you an API URL (e.g., https://git-platform-backend.onrender.com). Keep this URL handy.
🎨 Step 4: Deploy the Frontend on Vercel
Log into Vercel.com.
Click Add New > Project and import your GitHub repository.
In the build settings, change the Root Directory to frontend.
Leave the Build Command as npm run build and Output Directory as dist.
Under Environment Variables, add the environment variables to point to your backend.
Click Deploy. Vercel will assign you a frontend URL (e.g., https://git-platform.vercel.app).
🔄 Link Changes & Environment Variables (Crucial Step)
To connect all four components (Vercel, Render, Atlas, and Gitea), you do not need to edit the source code itself because the code already reads from environment variables!

You only need to supply the new production URLs and secrets in the Environment Variables panels of Vercel and Render.

1. In Vercel (Frontend Variables)
Add these environment variables inside your Vercel Project Settings:

Key	Value	Description
VITE_BACKEND_URL	https://git-platform-backend.onrender.com	Your backend Render URL
VITE_API_URL	https://git-platform-backend.onrender.com	Your backend Render URL (used for socket notifications)
VITE_GITEA_URL	https://gitea-production.up.railway.app	Your deployed Gitea URL on Railway
2. In Render (Backend Variables)
Add these variables inside your Render Web Service Environment panel:

Key	Value	Description
PORT	5000	Port assigned by host (Render handles this dynamically)
DB_URL	mongodb+srv://db_user:<password>@cluster0.xxxx.mongodb.net/github-style	Your MongoDB Atlas connection string
KEY	your_secure_random_jwt_secret	A long, secure random string for JWT encryption
CLIENT_URL	https://git-platform.vercel.app	Your deployed Vercel frontend URL
BACKEND_URL	https://git-platform-backend.onrender.com	Your deployed Render backend URL
GEMINI_API_KEY	AIzaSy...	Your Google Gemini API Key
CLOUDINARY_CLOUD_NAME	xxxx	Your Cloudinary Cloud Name
CLOUDINARY_API_KEY	xxxx	Your Cloudinary API Key
CLOUDINARY_API_SECRET	xxxx	Your Cloudinary API Secret
GITEA_BASE_URL	https://gitea-production.up.railway.app	Your deployed Gitea URL on Railway
GITEA_SSH_DOMAIN	gitea-production.up.railway.app (Remove https://)	Domain used for SSH repo URLs
GITEA_SSH_PORT	22 or 2222	Check your Gitea Railway configuration
GITEA_WEBHOOK_SECRET	MyCustomSecretPassword123!	Secure webhook verification phrase
GITEA_TOKEN_ENCRYPTION_KEY	SuperLongRandomStr4Encrypt537!32	A 32-character key for AES encryption
GITEA_ADMIN_TOKEN	See next step to generate	Gitea Admin token
🏁 Post-Deployment Sync (Final Setup)
Once the servers are online, you need to configure Gitea and connect it to your backend:

Initial Gitea Wizard: Open your Gitea Railway URL (https://gitea-production.up.railway.app) in your browser. Complete the initial installation steps (use SQLite or PostgreSQL database options presented by Railway). Create the first user, which automatically becomes the Admin User.
Generate Admin Token:
Log into your Gitea admin panel.
Go to Settings (top right) > Applications > Generate New Token.
Copy this token.
Add Token to Backend:
Go to your Render Backend dashboard.
Add the token as GITEA_ADMIN_TOKEN in the Environment Variables list.
Save and wait for Render to automatically redeploy your backend.
At this point, your MERN frontend, Express backend, MongoDB Atlas, and Gitea engine will be fully connected in production!
